# Backfill Discovery Digests with Expanded Category Set

## Context

The daily paper-discovery feed (`discovery/YYYY-MM-DD.md`) has been running since 2026-05-11.
Those digests were classified against only the **original 7** Library topics. Commit `b694d71`
added **8 new topics** (Joint Embedding Predictive Architectures, Multimodal Reasoning,
Open-Vocabulary Vision, Vision-Language Navigation, 3D Generation, 3D Reconstruction —
Feed-Forward, SLAM, Semantic Mapping) — both new BM25 keyword rows in
`.claude/skills/paper-discovery/SKILL.md` and new `Library/<Topic>/MOC.md` semantic specs.

Every historical digest is therefore stale: papers that should now surface under a new topic
were dumped into "Passed Over." The goal is to **re-classify every day in the range against the
full 15-topic set**, regenerating each `discovery/YYYY-MM-DD.md`.

Per user decisions:
- **Per-day, no dedup** — each digest covers exactly its own date (`Window: D → D`), classified
  independently. No cross-day dedup, no check against prior runs or the Library (nothing has been
  filed since launch).
- **Every calendar day** — generate a digest for every date 2026-05-11 → 2026-06-25 inclusive
  (45 days), including the ~5 gap days that never ran (05-14, 05-26, 06-08, 06-21, 06-22).

## Approach

Create a deterministic backfill script that mirrors `SKILL.md` Steps 2–8 for a single date and
loops the full range. Reuse the existing semantic classifier rather than reimplementing it.

### New file: `scripts/backfill_discovery.py`

Run under `researchEnv` (imports `sentence-transformers` via the existing classifier).

1. **Date range** — iterate every date from `2026-05-11` to `2026-06-25` inclusive.
2. **Fetch per date** — `curl -s "https://huggingface.co/api/daily_papers?date=D&limit=100&sort=publishedAt"`.
   Polite pacing (small sleep between calls) + a short retry on transient failure; tolerate empty
   responses. Use top-level `title`/`summary` for matching, `paper.id` for links,
   `paper.ai_summary or summary` for the blurb, `publishedAt` for the Published date. Dedup within
   a single date by `paper.id` (first occurrence).
3. **BM25 keyword map** — parse the `| Topic | Keywords |` table directly out of
   `.claude/skills/paper-discovery/SKILL.md` (single source of truth; backtick-wrapped,
   comma-separated). Case-insensitive substring match against `title + summary`.
4. **Semantic pass (once, not per-day)** — build the union of all unique papers across all 45
   dates and call `semantic_classify.classify(union, threshold=0.50)` **a single time** so the
   embedding model loads once. Cache `paper_id → {topic: score}`. (Import `classify` from
   `.claude/skills/paper-discovery/semantic_classify.py`.)
5. **Merge / best-fit** — per SKILL.md Step 5: a paper is a candidate for a topic only if it
   passes **both** BM25 and semantic; assign each paper to the single topic with the highest
   semantic score among its candidates; unmatched → Passed Over.
6. **Build digest** — exact current format (see `discovery/2026-06-24.md`):
   - Header: `# Paper Discovery — D`, then
     `> **Fetched:** D · **Window:** D → D` and the
     `> **Candidates:** N · **Prior duplicates skipped:** 0 · **Passed over:** P · **Matched:** M across T topics` line
     (prior-duplicates is always 0 in the no-dedup model).
   - Topic sections sorted by score desc; each entry: `### Title`,
     `**arXiv:** [id](https://arxiv.org/abs/id) | **Match:** 0.XX | **Published:** date`, blurb
     truncated to 2 sentences. No upvotes field (current format dropped it). No HF link.
   - `## Passed Over (P)` — titles only.
   - Empty day → `> No papers matched today's topics.`
   - Footer: `*Generated by paper-discovery skill · [HuggingFace Daily Papers](https://huggingface.co/papers)*`.
7. **Write** — overwrite `discovery/D.md` for each date.

### Topic-name alignment

Both the BM25 table and `semantic_classify` emit **canonical parent** names (e.g.
`Vision Language Action Models`), so the both-pass merge keys line up directly. Standalone
em-dash topics (`3D Reconstruction — Feed-Forward`) keep their full name on both sides.

## Critical files

- **Create** `scripts/backfill_discovery.py` — the backfill driver.
- **Reuse** `.claude/skills/paper-discovery/semantic_classify.py` → import `classify` (loads
  `all-MiniLM-L6-v2`, reads `Library/<Topic>/MOC.md`, folds sub-topics to parents).
- **Read-only source of truth** `.claude/skills/paper-discovery/SKILL.md` → BM25 keyword table +
  digest format spec.
- **Output (overwrite/create)** `discovery/2026-05-11.md` … `discovery/2026-06-25.md` (45 files).
- **Reference format** `discovery/2026-06-24.md`.
